# hermes-personality-rewrite

A Hermes agent plugin that intercepts final LLM outputs using the `transform_llm_output` hook and refines them using a smaller local LLM (such as Llama-3.1 8B or Gemma 2 9B) to fit the agent's personality anchor and purge AI slopisms.

## Installation

Clone this repository into your Hermes plugins directory:

```bash
cd ~/.hermes/plugins/
git clone https://github.com/thehiddenwolf/hermes-personality-rewrite.git personality-rewrite
```

Then enable it in your `~/.hermes/config.yaml` file:

```yaml
plugins:
  enabled:
    - personality-rewrite
```

## Configuration

### 1. Model Routing (`config.yaml`)
Register the auxiliary task under the `auxiliary:` key in your profile's `config.yaml`. For example, to route to a local uncensored model running on an isolated RTX 4090 port `11435`:

```yaml
auxiliary:
  personality_rewrite:
    provider: ollama
    model: huihui_ai/dolphin3-abliterated:8b-llama3.1-fp16
    base_url: http://127.0.0.1:11435
    api_key: ''
    timeout: 30
    extra_body: {}
```

### 2. Custom Files & Rules Override (`config.yaml`)
You can optionally configure the file paths for the personality refiner by adding a `personality_rewrite_plugin` block at the root of `config.yaml`:

```yaml
personality_rewrite_plugin:
  identity_anchor_file: "~/.hermes/profiles/eclipsia/config/identity_anchor.md"
  slop_counter_file: "~/.hermes/profiles/eclipsia/config/slop_counter.json"
  llm_config_file: "~/.hermes/profiles/eclipsia/config/rewrite_llm_config.json"
  system_prompt_file: "~/.hermes/profiles/eclipsia/config/rewrite_system_prompt.md"
```

If not specified, all files default to the active profile's config directory:
- **Default Profile (Spectra)**: `~/.spectra/config/`
- **Other Profiles (e.g., Eclipsia)**: `~/.hermes/profiles/eclipsia/config/`

## Configuration Files Explained

These files are auto-initialized on session start with sensible defaults if they don't exist:

### 1. `identity_anchor.md`
The character voice instructions. If missing, it automatically attempts to load and extract the anchor text from your profile's `SOUL.md` file (under the `## Identity Anchor` heading) or falls back to standard defaults.

### 2. `slop_counter.json`
Specifies rules and stores metrics for AI slop filtering:
- `slop_phrases`: An array of case-insensitive keywords/clichés to match and purge (e.g., `"certainly"`, `"i understand"`, `"delve"`).
- `counts`: A dictionary automatically updated by the plugin to keep track of how many times each phrase has been matched and stripped.

### 3. `rewrite_llm_config.json`
Custom LLM parameters passed to the auxiliary call (e.g. `temperature`, `max_tokens`, `timeout`, and custom parameters via `extra_body` like `top_p` or penalties).

### 4. `rewrite_system_prompt.md`
A SillyTavern-style system prompt template used to guide the rewrite process. It supports dynamic placeholder variables:
- `{{identity_anchor}}` — Injects the active character's anchor card.
- `{{slop_rules}}` — Injects formatting guidelines banning the slop phrases.
- `{{user_query}}` — Injects the user's original query (extracted from the active `SessionDB` database, allowing the refiner model to override cooperation levels, e.g., saying "No" to the user).
- `{{raw_draft}}` — Injects the raw output generated by the primary model.
- `{{detected_slops}}` — Injects the matched slops found in the current draft.
